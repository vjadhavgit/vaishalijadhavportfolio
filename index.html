01
AI PROGRAM LEADERSHIP · AZURE · MULTI-ORG
Multilingual Text-to-Speech AI Program
Georgia DDS + CITI + Horizons Innovation Lab · PoC complete · Pilot in progress
The problem

Live human interpreters for driver knowledge exams cost $160K+ annually. Manual scheduling created single points of failure. No audit trail. No equity of access in rural areas. No scalable path forward.

My decisions

Chose Azure over AWS — specifically because Azure supported Vietnamese, which Georgia's population required. AWS did not. Built the human translator evaluation framework. Redeployed the entire application solo to prove reproducibility. Led bias and Responsible AI review.

Where we are now

PoC complete — Spanish 89.6%, French 84% on weighted human evaluation. Pilot now in progress: testing audio feasibility live at one CSC center, collecting hardware compatibility data to identify which devices will reliably support audio file playback at scale before broader rollout.

Commerce Hub Payment Integration
Georgia DDS · CSC payment modernization · Handpicked for executive escalation leadership

// Payment architecture: token-based access flow · FS API integration · retry logic for expired tokens and denied payments

Leadership philosophy

How I lead.
In my own words.

When requirements are ambiguous, I don't wait.
The TTS PoC started with a vague brief and a tight timeline. Waiting for clarity is a decision — usually the wrong one. I build a working assumption, document it explicitly, get rapid stakeholder alignment, and move. Adjust as we learn. Standing still costs more than being slightly wrong.
I draw the RACI before anything else.
Every cross-functional program I've led had at least three departments pulling in different directions. Making ownership explicit — and sometimes uncomfortable — is what keeps momentum. Ambiguity in roles kills programs faster than technical complexity ever will.
Stakeholder resistance is usually about risk, not the idea.
The MDL Fast Lane was entirely new — no playbook, no precedent. Every objection was really "what if this fails?" I learned to reframe every pitch around what happens if we don't act. That shift changes the room faster than any slide deck.
ICs and senior leads need completely different things from me.
ICs need the "why" behind every decision — I over-communicate context. Senior leads need a sounding board and psychological safety to be wrong. The biggest mistake new managers make is giving both groups the same playbook. They're different problems.
I hold the line on quality gates — and make them lean.
The modular lockdowns on the HR Tracker were unpopular. Everyone wanted to move faster. I held the position because I knew what late-stage regression looks like in a system with three portals and live HR data. But I also made sure each gate was as lean as possible — not bureaucracy, just checkpoints.
A failure I actually learned from.
Early in the MDL Fast Lane rollout, I underestimated how long change management would take across 68 centers. The technical solution was right. The people timeline was wrong. Now I build change management effort explicitly into every program plan — not as an afterthought, as a parallel workstream.
